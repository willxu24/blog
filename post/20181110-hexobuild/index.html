写在前面

  • 为什么要花时间搭建自己的博客？
    • 个人想法的抒发空间
    • 自定义性非常强
    • 没有依赖于其他博客网站严格的审查与广告烦恼
    • 主要还是爱折腾
  • 搭建博客要做好的准备
    • 做好多次尝试，不断折腾的准备
    • 愿意花时间去慢慢摸索
    • 最后得出个人网站后成就感会很强
  • 主角简介
    • Hexo：一个快速、简洁且高效的博客框架。使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
    • GitHub：通过Git进行版本控制的软件源代码托管服务。
    • 博客构架简析(供纯小白比如博主这样的有助于理解)：工作架构是通过 git bash 创建博客本地文件夹,然后通过git上传到GitHub仓库，最后通过GitHub提供的gitpage功能来转换为网站展示。

一、本地博客环境搭建

1.准备软件：

  • 解决国内下载慢，两个软件的蓝奏云地址

密码:dj8i

  • Node.js：自主选择安装路径，注意图二选择，其余无脑下一步。

  • Git:可自主选择安装路径，注意图二的选择，博主选择第一个，其余无脑下一步。

2. hexo的安装

在桌面右键选择 git bash here

1

$ npm install -g hexo

安装完毕如图 ​

3. 博客文件夹初始化

a. 在本地任意你喜欢的合适位置创建博客本地文件夹（如博主D:\wrokspace\hexo） b. 在本地博客文件夹（之后的所有git bash）下右键进行git bash（等待时间较长，可以去上个洗手间或者刷一会儿微博）

1

$ hexo init


c. 生成文件夹如图所示

4. 博客内容的本地预览

本地博客文件夹下右键 git bash

1
2

$ hexo g # 生成public文件夹
$ hexo s # 启动本地服务

运行后打开图片中的网址 ​ 预览 ​

5. 本地环境搭建完毕

本地博客文件将会以md的形式储存在D:\wrokspace\hexo\source_posts 中

二、GitHub代码库搭建

1. 注册GitHub账号,创建新的仓库

2. 务必按照命名规范命名，并进行选项勾选。

未来登录自己博客的站点：https://用户名.github.io/

三、本地与GitHub通过git进行桥接

1. 配置用户名与邮箱

任意地方右键进行git bash

1
2

$ git config --global user.name "你的GitHub用户名"
$ git config --global user.email "你的GitHub注册邮箱"

成功后如图：

### 2. 配置ssh

a. 继续进行git bash

1

$ ssh-keygen -t rsa -C "你的GitHub注册邮箱"

b. 成功后如图

c. 继续进行git bash

1

$ clip < ~/.ssh/id_rsa.pub #复制ssh到剪贴板

d. 进入账户设置-ssh key设置-添加新ssh并保存

d. 进入账户设置-ssh key设置-添加新ssh并保存

3. 测试连接是否成功

继续进行git bash

1

$ ssh -T git@github.com #不要做更改

成功表现如图（中间会让你输入一次yes）

（这一步推荐用vscode或者其他文本编辑器，不推荐用win10自带的文本文档打开修改） 用vscode打开为例（红框为需要修改的部分）： 修改后： 其中的repository复制下图内容：

四、上传本地博客文件

1.博客文件夹下执行

1

$ npm install hexo-deployer-git --save #用于上传博客的插件

2.博客文件夹下执行

1
